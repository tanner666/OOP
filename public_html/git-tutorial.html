Setting up GitHub (Optional)

GitHub is a Git repository hosting service, but it adds many of its own features. While Git is a command line tool, GitHub provides a Web-based graphical interface. Github provides an easy way to host your repository online in order to isolate your work from the potential dangers of local machines and seamlessly collaborate with other developers.

Useful Commands

  • git init: Creates a new Git repository in the current working directory.
  • git status: Lists the current branch you are currently working on, as well as any which changes have been staged, which haven't, and which files aren't being tracked by Git.
  • git add <filename>: Stages the indicated filename to be ready for the next commit. Optionally, you can enter . in place of a filename to add all modified files in the current directory.
  • git commit -m "<commit-message>": Records stages changes to the repository. The -m tag indicates that there is a commit message.
  • git checkout <branch-name>: Switch working branches to <branch-name>. This allows you to text experimential features without comitting them to your main, working branch.
  • git checkout -b <branch-name>: Creates a new branch called <branch-name> and automatically switches to it.
  • git push <remote> <branch-name>: Pushes the local changes to the specified branch. If no branch is specified, it the default is remote origin.
  • git merge <branch-name>: Merges the named branch changes into the current working branch.
