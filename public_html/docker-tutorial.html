Docker Tutorial

What is Docker?

Docker is a platform for developing and testing applications in isolated environments, called containers, meaning that different applications won't interfere with one another. Click here for more information.

Installation

  1. Create a Docker Hub account.
  2. Install Docker Desktop.
    • Optional: Complete the "Getting Started" module.
  3. Troubleshooting: Open Docker Desktop. If an error occurs, try updating your operating system to the most current version. If this doesn't fix it, you may need to go into your system BIOS and enable virtualization. Click for a step-by-step guide.

Common Commands

  • docker build [OPTIONS] PATH | URL| -: Builds Docker images from a Dockerfile and a “context”. A build’s context is the set of files located in the specified PATH or URL.
  • docker run [OPTIONS] IMAGE[:TAG] [COMMAND] [ARG...]: Isolates a container process so that it has its own file system, its own networking, and its own isolated process tree separate from the host.
  • docker ps [OPTIONS]: Lists containers and useful information about them.
  • docker stop CONTAINER [CONTAINER...]: Stops one or more running container. You can use the container ID in place of the container.
  • docker rm [OPTIONS] CONTAINER [CONTAINER...]: Remove one or more containers. You can use the container ID in place of the container. Contianers must be stopped before than can be removed.
  • docker-compose up: Builds, (re)creates, starts, and attaches to containers for a service.
  • docker-compose down: Stops containers and removes containers, networks, volumes, and images created by docker-compose up.
