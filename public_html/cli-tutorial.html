Command Line Interface (CLI)


This tutorial goes over commands used in the Bash command line interface (CLI). This is different than Windows command prompt or Windows Powershell. Before you begin practicing these commands, be sure you have the Bash CLI. For macOS and Linux users, this should be native. Windows users should have Bash included in their Git download. If you don't have Git, and by extension Git Bash, installed, read our tutorial on Git.

Common Commands

  • pwd: Stands for present working directory. Returns the full path to the present directory.
  • cd <directory-name | directory-path>: Stands for change directory. Used to change the current working directory to the specified one.
    • Entering a .. in place of a path will take you back one directory. The . argument stands for the current directory, and each additional period goes back one directory.
    • The ~ key can be used to take you to the root directory of you PC.
  • ls: Lists the files and folders in a given directory.
  • touch <file-name>: Creates a file with the specified file name in the current directory.
  • mkdir <directory-name>: Creates a folder of the specified file name in the current directory.
  • rm <file-name>: Delete file.
    • In order to delete folders, you have to pass the -rf tag into your command: rm -rf <dir-name> The -r tag stands for recursive, meaning it will loop through the directory deleting all contents of the directory before it deletes the directory itself. The -f tag stands for force.
  • mv <old-name> <new-name>: Rename file <old-name> to <new-name>.
  • mv <file>... <directory>: Move the specified file(s) to the specified directory.
